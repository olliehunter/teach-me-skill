# MCPB Manifest.json Spec (modelcontextprotocol/mcpb)

URL: https://github.com/modelcontextprotocol/mcpb/blob/main/MANIFEST.md
Tier: 1 (primary — the authoritative MCPB specification in the MCP project)
Current version: 0.3. Last updated 2025-12-02. Accessed: 2026-07-01

## Overview
"The manifest.json file contains all extension metadata and configuration. Most fields are optional."

## Minimal required manifest
```json
{
  "manifest_version": "0.3",
  "name": "my-extension",
  "version": "1.0.0",
  "description": "A simple MCP extension",
  "author": { "name": "Extension Author" },
  "server": {
    "type": "node",
    "entry_point": "server/index.js",
    "mcp_config": {
      "command": "node",
      "args": ["${__dirname}/server/index.js"]
    }
  }
}
```

### Required fields
- **manifest_version**: spec version this bundle conforms to.
- **name**: machine-readable name (used for CLI, APIs).
- **version**: semantic version (semver).
- **description**: brief description.
- **author**: object with `name` (required), optional `email`/`url`.
- **server**: server configuration object.

### Notable optional fields
`display_name`, `long_description`, `repository`, `homepage`, `documentation`, `support`, `icon`/`icons`,
`screenshots`, `tools` (array), `tools_generated` (bool), `prompts` (array), `prompts_generated`,
`keywords`, `license`, `privacy_policies` (required when connecting to external services that process
user data), `compatibility`, `user_config`, `localization`, `_meta`.

## Server configuration & types
The `server` object defines how to run the MCP server. Four server types:
- **node** — Node.js server with bundled dependencies (in `node_modules`).
- **python** — Python server; all dependencies bundled (`server/lib` or `server/venv`).
- **binary** — pre-compiled, self-contained executable (no runtime requirement).
- **uv** (v0.4+) — Python via UV runtime; dependencies declared in `pyproject.toml` and installed by
  the host, NOT bundled. Benefits: cross-platform, small bundle (~100 KB vs 5–10 MB), no user Python
  install required. Must NOT include `server/lib/` or `server/venv/`.

### mcp_config
"The mcp_config object... defines how the implementing app should execute the MCP server. This
replaces the manual JSON configuration users currently need to write." Has `command`, `args`, `env`,
optional `platform_overrides` (per `win32`/`darwin`/`linux`).

### Variable substitution
- `${__dirname}` — absolute path to the extension's installed directory.
- `${HOME}`, `${DESKTOP}`, `${DOCUMENTS}`, `${DOWNLOADS}` — user directories.
- `${pathSeparator}` / `${/}` — platform path separator.
- `${user_config.KEY}` — a user-provided configuration value.
"This ensures that paths work correctly regardless of where the extension is installed."

## User configuration
"The user_config field allows extension developers to specify configuration options... collected from
users and passed to the MCP server at runtime." Each option has:
- `type`: `string`, `number`, `boolean`, `directory`, or `file`.
- `title`, `description`.
- `required` (default false), `default` (supports variable substitution).
- `multiple` (directory/file — allow multiple; expanded as separate args).
- `sensitive` (string — mask input and store securely; ideal for API keys, stored in OS keychain).
- `min`/`max` (number validation).

Example — API key passed as env var:
```json
"user_config": {
  "api_key": { "type": "string", "title": "API Key", "sensitive": true, "required": true }
},
"server": { "mcp_config": { "env": { "API_KEY": "${user_config.api_key}" } } }
```

## Compatibility
`compatibility` (all optional): client version constraints (e.g. `claude_desktop: ">=1.0.0"` — semver),
`platforms` (`darwin` = macOS, `win32` = Windows, `linux`), `runtimes` (e.g. `python: ">=3.8"`,
`node: ">=16.0.0"` — specify only the runtime your server uses; binaries need none).

## Tools and prompts declaration
Static: list `tools`/`prompts` arrays. Dynamic: set `tools_generated: true` / `prompts_generated: true`
to indicate the server generates more at runtime. "Resources are not included in the manifest because
MCP resources are inherently dynamic."
